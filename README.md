# Тестовое задание: Middle ML Engineer
## Проект: Система динамического ценообразования и анализа аномалий

## Решение

Основной файл решения — `Middle_MLE_HW.ipynb`. Код разбит по предметным
модулям пакета `retail_pricing/`:

- `data.py` — загрузка и аудит качества данных;
- `anomaly.py` — детекция аномалий в `sales` и оценка детектора;
- `features.py` — причинные лаги и признаки спроса;
- `demand.py` — модель спроса и временная валидация;
- `explainability.py` — факторный анализ и root cause report;
- `elasticity.py` — оценка и регуляризация ценовой эластичности;
- `optimization.py` — целевые функции, ограничения и подбор цены.

Реализованы обе части задания:

1. Детекция аномалий и определение факторов, формирующих ожидаемый спрос.
2. Прогноз спроса на следующий день и подбор цены, максимизирующей
   `(Цена - Себестоимость) × Продажи`. Дополнительно показаны оптимумы для
   продаж и GMV, чтобы явно продемонстрировать конфликт бизнес-целей.

### Архитектурные решения

- Детектор использует только прошлые значения каждого временного ряда и
  разделяет high-confidence технические алерты и более широкий review-слой.
- Глобальная XGBoost-модель обучается сразу на всех товарах. Валидация
  выполняется expanding-window фолдами без перемешивания будущего и прошлого.
- Root cause строится встроенным TreeSHAP XGBoost и отдельно учитывает
  необъяснённый остаток прогноза.
- Эластичность регуляризуется к prior категории. Для ненадёжных индивидуальных
  оценок изменение цены ограничено 5%, для подтверждённых — 15%.
- Оптимизация выполняется прозрачным grid search с ограничениями по марже,
  величине изменения и историческому диапазону цен.

- Offline-оценка ценовой эластичности на наблюдательных данных не доказывает

  как в исходных данных продажи часто превышают указанный остаток.

## 📈 Контекст задачи

Вы — ML-инженер в отделе автоматизации ценообразования. Департамент переходит на алгоритмическое управление. В вашем распоряжении исторические данные о продажах 200 товаров за последние 2 года. 

**Ваша цель:** Разработать прототип системы, которая умеет находить сбои в данных и оптимизировать цены для максимизации прибыли. 

### Описание данных и источники
Основной датасет: `retail_mle_task_data.parquet`.

Признаки (прим. детали в описании внутри тетрадки):
*   **date**: Дата наблюдения.
*   **item_id**: Идентификатор товара.
*   **category**: Категория товара.
*   **price**: Текущая цена товара.
*   **cost**: Себестоимость товара.
*   **sales**: Количество проданных единиц (целевая переменная).
*   **promo_active**: Флаг активной промо-акции (0 или 1).
*   **competitor_price**: Цена аналогичного товара у конкурента.
*   **ad_spend**: Затраты на рекламу.
*   **stock_level**: Уровень остатков на складе.
*   **weather_index**: Индекс погодных условий.

**Дополнительные данные:** Вы можете генерировать собственные признаки на основе имеющихся (лаги, скользящие средние, праздники и т.д.) или использовать открытые внешние источники, если считаете это необходимым.

## 📝 Практическое задание

Задание состоит из двух функциональных частей, которые необходимо реализовать в коде.

### Часть 1: Детекция аномалий и факторный анализ
Бизнесу нужно отличать реальные рыночные всплески от технических ошибок в данных.

1.  **Поиск аномалий:** Реализуйте алгоритм для поиска аномальных значений в колонке `sales`. 
2.  **Факторный анализ (Root Cause):** Предложите способ (или реализуйте прототип), позволяющий понять, какой признак внес наибольший вклад в возникновение аномалии.

### Часть 2: Динамическое ценообразование (Optimization)
Нужно предложить цены на следующий день, которые максимизируют продажи и прибыль (GMV).

1.  **Моделирование спроса:** Постройте модель, предсказывающую объем продаж в зависимости от цены и других факторов.
2.  **Оптимизация:** Разработайте алгоритм, который на основе модели спроса подбирает оптимальную цену для максимизации целевой функции (Прибыль = (Цена - Себестоимость) * Продажи).
